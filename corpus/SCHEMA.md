# Morpheus Index Schema (`corpus/morpheus.db`)

SQLite database produced by `scripts/build_index_db.py` from
`corpus/unified.jsonl` (canonical paper list) and
`papers/arxiv/MANIFEST.jsonl` (PDF download status).

Rebuild with:

```bash
python3 scripts/build_index_db.py --rebuild
```

## Tables

### `papers`
One row per logical paper. `arxiv_id` is `NULL` for code-only / non-arXiv entries.

| column             | notes                                                          |
| ------------------ | -------------------------------------------------------------- |
| `paper_id`         | PK                                                             |
| `arxiv_id`         | unique; `NULL` allowed (~41 code-only rows)                    |
| `title`            | required                                                       |
| `year`             | parsed as INTEGER (best-effort from unified.jsonl)             |
| `abstract`         | empty on load — populated later from PDF ingest                |
| `primary_category` | arXiv primary category; empty on load (not in unified.jsonl)   |
| `github_url`       | first entry of `github_urls` if present                        |
| `suspicious`       | 0/1, mirrors unified flag                                      |
| `created_at`       | load timestamp                                                 |

### `authors`, `paper_authors`
Empty at build time. A separate enrichment agent fills these from
arXiv metadata. `authors.name_normalized` is the dedup key (lowercased,
diacritic-stripped). `paper_authors.position` preserves author order.

### `tags`, `paper_tags`
Flexible polymorphic taxonomy. `tags.kind` uses this controlled vocabulary:

| kind       | meaning                                                  |
| ---------- | -------------------------------------------------------- |
| `area`     | broad research area (e.g. "agentic-rl", "tool-use")      |
| `subject`  | finer-grained topic (e.g. "multi-agent coordination")    |
| `method`   | technique (e.g. "PPO", "GRPO", "reward-model")           |
| `artifact` | produced artifact (e.g. "benchmark", "dataset", "code")  |
| `category` | arXiv primary_category (populated when available)        |

`paper_tags.confidence` is `1.0` for manually curated tags and can be
lowered for tags inferred by LLM passes.

### `downloads`
One row per paper with a PDF on disk. `status` is `downloaded` | `failed` |
`missing`. `path` is relative to the repo root, e.g. `papers/arxiv/2509.02547.pdf`.
Loader normalizes manifest `pre_existing` status to `downloaded`.

### `fulltext` (FTS5 virtual)
`paper_id, title, abstract, body`. Empty at build time — populated by a
later PDF-extraction pass. Use contentless (`content=''`) FTS5, so inserts
must set `rowid=paper_id`.

## Example queries

**1. Papers by a given author**

```sql
SELECT p.arxiv_id, p.year, p.title
FROM papers p
JOIN paper_authors pa USING(paper_id)
JOIN authors a USING(author_id)
WHERE a.name_normalized = 'yann lecun'
ORDER BY p.year DESC;
```

**2. Papers with a given tag**

```sql
SELECT p.arxiv_id, p.title
FROM papers p
JOIN paper_tags pt USING(paper_id)
JOIN tags t USING(tag_id)
WHERE t.kind = 'method' AND t.name = 'GRPO';
```

**3. Papers missing PDFs**

```sql
SELECT p.arxiv_id, p.title
FROM papers p
LEFT JOIN downloads d USING(paper_id)
WHERE p.arxiv_id IS NOT NULL
  AND (d.status IS NULL OR d.status <> 'downloaded');
```

**4. Papers by year + area**

```sql
SELECT p.year, COUNT(*) AS n
FROM papers p
JOIN paper_tags pt USING(paper_id)
JOIN tags t USING(tag_id)
WHERE t.kind = 'area' AND t.name = 'agentic-rl'
GROUP BY p.year
ORDER BY p.year;
```

**5. Full-text search (available after PDF ingest)**

```sql
SELECT p.arxiv_id, p.title
FROM fulltext f
JOIN papers p ON p.paper_id = f.paper_id
WHERE fulltext MATCH 'reward hacking AND GRPO'
ORDER BY rank
LIMIT 20;
```
